# Tsunagu — シナリオ別 画面遷移と表示情報

## 0. このドキュメントの位置づけ

`.designs/_pre/00-foundation.md` の機能リストを、**実装で迷う余地のない画面シーケンス**に落とし込む。
Architectは本書を入力として、データモデル/API/画面コンポーネント仕様を執筆する。

各画面は「上 (30%) / 中 (60%) / 下 (10%)」の3ペイン構造で記述。

---

## 1. デザインシステム原則 (再掲)

| 原則 | 具体 |
|---|---|
| 主要素1個 | 画面の60%は単一要素 (SOSボタン / 症状選択 / CPR音声) |
| テキスト12文字以下 | 「押して」「呼吸なし」レベル |
| フォント24pt以上 | 主要素は40pt+ |
| コントラスト 7:1 (WCAG AAA) | 黒地+白文字+赤アクセント |
| タップ領域 88pt以上 | iOSデフォルトの2倍 |
| 1画面 1〜3選択 | 多くて4 |
| 音声優先 | CPRは声、画面は確認のみ |
| 戻る/キャンセルは隠す | 緊急画面では存在しない、長押しのみ |
| 進捗常時表示 | 「AEDあと2分10秒」など不確実性を消す |
| 罪悪感ゼロ | キャンセル・撤退OK |

### カラーパレット

```
緊急 (Emergency Red):    #DC2626
情報 (Info Blue):        #2563EB
成功 (Success Green):    #16A34A
背景 (Black):            #0A0A0A
前景 (White):            #FAFAFA
副 (Gray-700):           #404040
警告 (Amber):            #F59E0B
```

### タイポ

```
H1: 40pt SemiBold (主要素)
H2: 28pt SemiBold (見出し)
Body: 24pt Regular (説明)
Small: 18pt Regular (補足、最小)
```

## 3. メインシナリオ F1: 発見者が「呼吸なし」を選択 (心停止疑い)

### 概要
- 目的: AEDを届け、CPRを成立させ、生存率を上げる
- 想定時間: 起動→引き継ぎまで 3〜5分
- 関連画面: HOME → CD → SYM → RM-F → (AED-G) → HOFF → END

### 3.1 画面: HOME (ホーム / 待機)

```
┌─────────────────────────┐
│  ⚙ (右上隅、極小)        │ ← 設定アイコン (タップ領域88pt)
│                         │
│                         │
│      ┌───────────┐      │
│      │           │      │
│      │   SOS     │      │ ← 中央 (画面の60%、丸ボタン直径280pt)
│      │           │      │   背景: #DC2626 (Emergency Red)
│      │  ヘルプ   │      │   文字: 白 32pt SemiBold
│      │           │      │
│      └───────────┘      │
│                         │
│                         │
│  最寄AEDマップを見る ↗  │ ← 下 (副アクション、24pt)
└─────────────────────────┘
```

| ペイン | 表示要素 | タップ動作 |
|---|---|---|
| 上 (30%) | 設定アイコン (右上極小) | プロフィール画面 (PROFILE) へ |
| 中 (60%) | 巨大な赤い円型SOSボタン「ヘルプ」 | CD画面へ |
| 下 (10%) | 「最寄AEDマップを見る ↗」 | AEDマップ単独画面 (緊急じゃない閲覧) |

異常時:
- 位置情報未許可 → 中央ボタンタップでPERM画面
- ネット切断 → ボタン下に小さく「オフライン: 119コールのみ可能」

---

### 3.2 画面: CD (カウントダウンキャンセル)

```
┌─────────────────────────┐
│                         │
│                         │
│           3             │ ← 巨大数字 200pt (3秒→2→1)
│                         │
│       ヘルプを呼びます    │ ← 28pt
│                         │
│                         │
│   画面を長押しでキャンセル  │ ← 24pt 下部
│                         │
└─────────────────────────┘
```

| ペイン | 表示要素 | アクション |
|---|---|---|
| 中 | 「3」「2」「1」の巨大カウント | 1秒ごと減算 |
| 中下 | 「ヘルプを呼びます」 | 静止 |
| 下 | 「画面を長押しでキャンセル」 | 長押し2秒でHOMEへ戻る (ペナルティなし) |

カウント0で自動的にSYM画面へ遷移。

### 3.3 画面: SYM (症状選択)

```
┌─────────────────────────┐
│  どんな状況ですか？       │ ← 28pt 上部
├─────────────────────────┤
│ ┌─────────┐ ┌─────────┐ │
│ │  🫁❌    │ │  💤❌    │ │ ← 4アイコン、各160x160pt
│ │ 呼吸なし│ │ 意識なし│ │
│ └─────────┘ └─────────┘ │
│ ┌─────────┐ ┌─────────┐ │
│ │  🩸     │ │  ❓     │ │
│ │  出血   │ │ その他   │ │
│ └─────────┘ └─────────┘ │
├─────────────────────────┤
│  ━━━━━━━━━━━━━━━━━━━━   │ ← スワイプエリア (高さ80pt)
│  119を呼ぶ →             │   24pt 白文字、赤背景
└─────────────────────────┘
```

| ペイン | 表示要素 | アクション |
|---|---|---|
| 上 (30%) | 「どんな状況ですか？」 | - |
| 中 (60%) | 4アイコン (呼吸なし/意識なし/出血/その他) | タップで選択、即発火 |
| 下 (10%) | 「119を呼ぶ →」スワイプエリア | 右にスワイプで `tel:119` (デモは`tel:09039655913`) 起動 |

選択後の挙動:
- **呼吸なし → RM-F画面 (CPRガイド+AED運搬通知)**
- 意識なし → RM-F画面 (回復体位ガイド版、F2、MVPでは同じRM-Fを「CPRはせず体位確認」モードで)
- 出血 → RM-F画面 (止血ガイド版、F3、同上)
- その他 → RM-F画面 (Q助 deep linkを表示、F4)

MVPでは4選択肢全てRM-Fに遷移、画面内コンテンツが症状で変わる。F1 (呼吸なし) のみフルCPR音声+AED通知、他は簡易ガイド+救助者通知のみ。

サーバ送信:
- 選択直後に `POST /sos` (位置・症状・現場プロファイル送信)
- 「呼吸なし」の場合のみ AED運搬通知が即時配信

---

### 3.4 画面: RM-F (救助モード - 発見者用)

```
┌─────────────────────────┐
│ 🟢 AEDあと 2分10秒       │ ← 上 30% モチベ表示
│ 🔵 救助者あと 3分30秒    │   各24pt、左寄せ
├─────────────────────────┤
│                         │
│    ♥ 押して ♥ 押して     │ ← 中 60% メイン
│                         │   40pt SemiBold
│   ━━━━━━━━━━━━━━       │   メトロノーム視覚 (脈動)
│                         │   音声: 「続けて、いい感じ」
│   100 BPM (1分100回)    │   18pt 副情報
│                         │
├─────────────────────────┤
│ 119を再度呼ぶ            │ ← 下 10% 副アクション
│ 場所: マンション3F      │
└─────────────────────────┘
```

| ペイン | 表示要素 | 詳細 |
|---|---|---|
| 上 (30%) | AED到着までの時間 + 救助者到着までの時間 | サーバから5秒間隔で更新、見える化が最重要 |
| 中 (60%) | 「押して」+ メトロノーム視覚 + 音声 | 100-120BPM、JRC公式CPRガイドライン準拠、「ターン、ターン...」音声 |
| 下 (10%) | 119再コール + 現在地 | 119はいつでも再発火可、現場プロファイル (住所/階数) は救助者にも送信 |

音声LLMの役割 (重要):
- メトロノーム音 (リズム維持)
- 励まし音声「いい感じ、続けて」「あと1分で応援が来ます」 (30秒に1回)
- 「強く、速く、深く5cm押して」のキュー (1分に1回)
- AED到着時 → 「AED来ました、装着してください」音声+AED-G画面に自動遷移

サーバとの通信:
- WebSocket で `/ws/rescue/<rescueId>` に接続
- 救助者の応答状況、AED運搬役の現在位置を購読

異常時:
- ネット切断 → 上部に「オフライン中、CPRを続けて」表示、CPR音声ガイドだけは継続
- AED運搬役ゼロ → 「AEDなし、CPRを続けて」表示、AED時間表示は非表示

### 3.5 画面: AED-G (AED装着ガイド)

```
┌─────────────────────────┐
│ ① 電源を入れる           │ ← 上 進捗 1/4
│ ━━━━━━━━━━━━━━━━━━━━   │   プログレスバー
├─────────────────────────┤
│                         │
│  [AED画像 / 動画 large]   │ ← 中 60% ビジュアル
│                         │
│   電源ボタンを押す        │   28pt
│                         │
├─────────────────────────┤
│ AEDの音声に従ってください │ ← 下 注意書き
└─────────────────────────┘
```

ステップ4段階:
1. 電源ON
2. パッドの貼り方 (胸に2枚、絵で示す)
3. 解析中 (患者から離れる)
4. ショック or 圧迫継続 (AED本体の音声指示に従う)

| ペイン | 表示要素 | アクション |
|---|---|---|
| 上 | 「① 電源を入れる」+ プログレスバー | 4ステップで進む |
| 中 (60%) | AED画像 (大) + 短い指示文 | 各ステップで切り替え、画像はAED N@VIで紐づく機種別 |
| 下 | 「AEDの音声に従ってください」 | 静的注意書き |

進行: 各ステップは「次へ」ボタン or 自動遷移 (実機AEDの動作と同期できないので手動進行)

```
│      田中 (医学生)       │   40pt
│                         │
│   引き継ぎますか？        │   28pt
│                         │
├─────────────────────────┤

```

| ペイン | 表示要素 | アクション |
|---|---|---|
| 上 | 「{Tier名}さんが到着しました」 | 救助者名表示 |
| 中 (60%) | 救助者名+Tier+アバター | 信頼の可視化 |
| 下 (10%) | 「引き継ぐ」「続ける」2択 | 引き継ぐ→RM-Rに切替 / 続ける→救助者にCPR交代しない |

「引き継ぐ」を選ぶと:
- 発見者の画面: 「あなたの行動が命を救った」+ サポート役モードに移行
- 救助者の画面: RM-R に切り替わり、CPR音声ガイドを継承

- 統計用の匿名データ (発火→現着→引き継ぎ時間, AED使用有無) のみ永続化

## 4. 救助者シナリオ R系 (Tier1: 医療者 / Tier2: 学生)

### 4.1 画面: NOTI-R (救助者通知画面 / R1: 通知受信)

ロック画面通知 (PWAではWeb Push、ネイティブではAPNs/FCM Critical Alert):

```
┌─────────────────────────┐
│ Tsunagu — 緊急ヘルプ要請  │ ← Notification title
├─────────────────────────┤
│ 350m 先・心停止疑い       │ ← Body text
│ 「呼吸なし」マンション3F   │
├─────────────────────────┤
│ [向かう] [手ぶらで] [無理] │ ← 3 actions
└─────────────────────────┘
```

3択アクション:
- **「向かう」(AED持って向かう)** → アプリ起動 → NAV-R画面 + AEDをルート上で取得
- **「手ぶらで」** → アプリ起動 → NAV-R画面 (AEDは別の人が運ぶ前提)
- **「無理」** → R2画面 (キャンセル)

iOS Critical Alert は要Apple認可。PWAでは通常のWeb Push。MVPはWeb Pushで開始、後でCritical Alert申請。

---

### 4.2 画面: NAV-R (救助者ナビ / R1: 向かう途中)

```
┌─────────────────────────┐
│ ⏱ 残り 3分20秒           │ ← 上 30% モチベ
│ 350m 先 マンション3F     │   24pt
├─────────────────────────┤
│                         │
│  [マップ表示 large]       │ ← 中 60% ナビマップ
│   現在地 → 現場          │   経路ライン (青)
│   AED運搬役の現在位置 (緑) │   他救助者の位置 (黒)
│                         │
├─────────────────────────┤
│ 症状: 呼吸なし、意識なし   │ ← 下 10% 状況情報
│ [諦める]                  │   ← 撤退ボタン (長押し)
└─────────────────────────┘
```

| ペイン | 表示要素 | アクション |
|---|---|---|
| 上 (30%) | 残り時間 (推定到着) + 現場ラベル | リアルタイム更新 |
| 中 (60%) | マップ (現在地→現場、AED運搬役の位置同時表示) | 経路はApple Maps / Google Maps API |
| 下 (10%) | 症状情報 + 諦めるボタン (長押し2秒) | 諦めるはR3シナリオへ |

向かう途中の救助者が見える理由: 自分以外に誰が向かってるか可視化することで「もう来る人いるな」(=自分が無理しなくていい) の判断ができる + 二重通知の連帯感。

---

### 4.3 R2: 「無理」を選択 (撤退)

通知から「無理」を押した、または NAV-R で「諦める」を押した時:

```
┌─────────────────────────┐
│   お疲れ様でした           │ ← 上 28pt
├─────────────────────────┤
│                         │
│   今回は対応できなくて      │ ← 中 60%
│   大丈夫です              │   28pt
│                         │
│   他の救助者が向かっています │   24pt
│   (現在 2人)              │
│                         │
├─────────────────────────┤
│ [閉じる]                  │ ← 下
└─────────────────────────┘
```

罪悪感ゼロ設計:
- 「対応できなくて大丈夫です」と明示
- 他の救助者がいる事実を伝える (緩衝)
- 通知履歴には残さない (「あなたは無理だった」と再表示しない)

### 4.4 画面: RM-R (救助モード - 救助者用 / R4: CPR交代)

引き継ぎ完了後、救助者の画面に出る:

```
┌─────────────────────────┐
│ 🟢 救急隊あと 4分20秒    │ ← 上 30%
│ 患者: 男性, 約60代        │
├─────────────────────────┤
│                         │
│    ♥ 押して ♥ 押して     │ ← 中 60% (発見者と同じ)
│   ━━━━━━━━━━━━━━       │   メトロノーム
│                         │
│   100 BPM (1分100回)    │
│                         │
├─────────────────────────┤
│ 引き継ぎ前の経過: 3分    │ ← 下 状況
│ AED装着状況: 装着済        │
└─────────────────────────┘
```

発見者用と異なる点:
- 上部の表示が「救急隊あと」(救助者は救急隊に引き継ぐのが目的)
- 下部に経過情報 (「すでに3分CPRされてる」など現状把握)
- 患者の見た目情報 (発見者が選択肢で送信、または救助者が見て即時記録)

### 4.5 画面: AED-G (R5: AED装着、救助者として)

発見者と共通画面、表示は同じ。
ただし救助者がAED-Gにいる時、発見者はRM-Fで待機 or サポート役。

### 4.6 R6: 救急隊到着→引き継ぎ

HOFF画面の救急隊版:

```
┌─────────────────────────┐
│   救急隊員が到着しました    │
├─────────────────────────┤
│   ハンドオフを完了して      │
│   すぐ離れてください       │
│                         │
│   救急隊が処置を引き継ぎます │
```

## 5. AED運搬シナリオ A系 (Tier3)

### 5.1 画面: NOTI-A (AED運搬通知画面 / A1)

ロック画面通知:

```
┌─────────────────────────┐
│ Tsunagu — AED運搬要請     │
├─────────────────────────┤
```

## 10. 状態遷移とサーバ通信 (簡易)

### サーバ送信イベント
- `POST /sos` — SOS発火、症状・位置・現場プロファイル
- `POST /rescue/<id>/respond` — 救助者の応答 (向かう/無理)
- `WS /ws/rescue/<id>` — リアルタイム位置共有・状態更新
- `POST /rescue/<id>/handoff` — 引き継ぎ完了
- `POST /rescue/<id>/end` — 終了

### サーバ受信イベント (発見者→受信)
- `responder_assigned` — 救助者が応答した
- `aed_carrier_assigned` — AED運搬役が応答した
- `responder_eta_update` — 到着時間更新 (5秒間隔)
- `responder_arrived` — 救助者到着
- `aed_arrived` — AED到着 → AED-G画面に遷移
