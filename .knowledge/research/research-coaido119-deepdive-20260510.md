## 1. サービスの軌跡

### 1.1 創業・初期
- **2014年**: 玄正慎 (げんしょう まこと) 氏が、ハッカソンで心停止救助アプリ「AED SOS」を作って優勝したのを機に Coaido株式会社を設立。京都大学との共同研究開始 (2014-08-08)。
  - 出典: [Startup DB - Coaido](https://startup-db.com/companies/vjwKJ2MUbQDZmQP1) / [TechCrunch Japan 2015-08-31](https://jp.techcrunch.com/2015/08/31/coaido/) (※TLS証明書エラーで本文未取得、検索結果に存在は確認)
- **2017年**: 緊急情報共有アプリ「Coaido119」を開発、経済産業省「第3回 IoT Lab Selection」グランプリ受賞。
- **2017-08〜2018-01**: 東京都豊島区(池袋駅周辺)で実証実験 (6ヶ月)。
  - 出典: [WirelessWire 2017-09](https://wirelesswire.jp/2017/09/61192/)
- **2017-10-26**: 一般利用者登録開始。
- **2017-12-20**: 関連法人「**一般社団法人ファストエイド**」設立 (代表理事も玄正氏が兼務)。
  - 出典: [CANPAN - 一般社団法人ファストエイド](https://fields.canpan.info/organization/detail/1282961224)
- **2018-02-26**: 全国対応開始。AED情報は163自治体56,721件を実装。
  - 出典: [週刊アスキー 2018-03-02](https://weekly.ascii.jp/elem/000/001/641/1641216/) / [ITmedia Mobile 2018-02-28](https://www.itmedia.co.jp/mobile/articles/1802/28/news124.html)

## 2. 機能と仕組み

### 2.1 SOS発火フロー
- ユーザー(発見者)が**119番にダイヤルしながら**アプリのSOSボタンを押す → 位置情報・状況がプッシュ配信される。
- セルフトリアージにより緊急度を判定する設計。
  - 出典: [INTERNET Watch (地図と位置情報)](https://internet.watch.impress.co.jp/docs/column/chizu3/1100451.html)

### 2.2 受け手 (受信側)
事前登録した以下の人にSOSが届く:
- 医療有資格者 (医師・看護師・救急救命士など)
- 救命講習受講者 (CPR訓練修了者)
- AED設置施設の安全管理者 (受信専用、駆けつけはしない)
- iOS のみ対応 (★Androidユーザーは受信側登録不可だった)
  - 出典: [Coaido119 FAQ (検索結果より)](https://www.coaido119.com/faq) / [App Storeレビュー](https://applion.jp/iphone/app/1192291275/review/)
- ★**18歳未満の登録不可**(救命講習修了者であっても)
  - 出典: [App Storeレビュー](https://applion.jp/iphone/app/1192291275/review/)

### 2.3 「AEDエリアコール」
- SOS発信地から**半径300m / 最大10施設**のAED設置施設の電話に**自動架電**し、AED運搬を依頼。
  - 出典: 検索サマリ複数 (Coaido119 公式 / [risktaisaku.com](https://www.risktaisaku.com/articles/-/4022))

### 2.4 119連携
- ★**消防の指令センターと直接連携する仕組みではなかった**。「ユーザー側で119番にダイヤルしつつ、並行してアプリでSOSを撒く」という二段構え。
  - これは AED GO! (株式会社ドーン / 京都大学) や PulsePoint Respond (米) と決定的に違う点。
  - 出典: [INTERNET Watch](https://internet.watch.impress.co.jp/docs/column/chizu3/1100451.html) / [Coaido119 FAQ](https://www.coaido119.com/faq)

### 2.5 救助実績の検証
- 2017-2018のテスト期では、**実際の心停止現場で発火・救命された明確な「1号」事例の報道は確認できず**、ドリル(訓練)での有効性検証が中心だった。
  - 出典: 検索結果サマリ (Coaido119 関連記事横断)
- [不確定] 2019年以降の実救命件数の公式統計は本調査では発見できず。

## 3. 終了原因の分析

### 3.1 a. ユーザー獲得 ★
- **発見者(SOS発火側)の登録**: 全国対応後 TV露出で1万到達。ただし**アクティブ密度はエリア依存**で、池袋以外では「周りに登録者がいない」状態が続いた。
  - 出典: [App Storeレビュー](https://applion.jp/iphone/app/1192291275/review/) — "周りでインストールしてる人がいない"
- **救助者(受信側)の登録**: 医療従事者・CPR講習修了者の絶対数が日本国内で限られる上、**18歳未満NG・iOS限定** という受信側の入口の狭さが致命的。
  - 出典: 同上
- ★**Two-sided market の典型詰み**: 発見者・救助者の両側を**同時に同一エリアで密度高く揃える必要がある**が、無料アプリ広告予算ゼロでは実現困難。

### 3.2 b. 救助実績
- 訓練・デモでは「ほぼ確実に現場特定可能」「動画情報が強力」と報告。
  - 出典: [マイナビニュース 2017-11-10](https://news.mynavi.jp/article/20171110-coaido119/2) / [Mac Fan](https://macfan.book.mynavi.jp/article/m89153/)
- ★**実際の心停止現場で発火された SOS の応答率・救命寄与率の数字が公開されない**ままサービス終了。実績データの不在は次の自治体提携・資金調達を阻む大きな要因と推測。

### 3.3 c. 収益モデル ★
- 利用者向けは完全無料。一般ユーザー登録、SOS送受信、救急相談ダイヤル検索すべて無料。
  - 出典: [Coaido119 FAQ](https://www.coaido119.com/faq)
- 玄正氏自身が「**無料でも価値があれば広がり、いずれビジネスになる**」と発言 → 後追いマネタイズ前提の見切り発車。
  - 出典: [INTERNET Watch](https://internet.watch.impress.co.jp/docs/column/chizu3/1100451.html)
- 構想していた将来課金: 生命保険会社協業 / マンション(タワーマンションの救急遅延対策) 向けサービス / ウェアラブルセンサー連動の自動通報。**いずれも構想段階で実装に至った形跡なし**。
- ★Coaido119単体での課金チャネルがなく、**B2C無料 → B2G/B2Bピボットを成功させる前にランウェイ切れ**となった構図。

### 3.4 d. 行政連携 ★
- 自治体提携の中核は**東京都豊島区**ただ1ヶ所。SECOM協業もこのエリアに局所化。
  - 出典: [WirelessWire](https://wirelesswire.jp/2017/09/61192/)
- 玄正氏発言: 「いきなり全国各地のAED設置施設に連絡を取って了解を得るのは難しい」 — AEDエリアコールの全国展開には**施設ごとの個別合意が必要**で、スケールの構造的ハンデ。
  - 出典: [INTERNET Watch](https://internet.watch.impress.co.jp/docs/column/chizu3/1100451.html)
- ★**消防の指令センターとの統合がない** → 「公式に位置づけられた救命インフラ」になれず、自治体予算化(B2G)の根拠を作りにくかった。
  - 比較対照: AED GO!(株式会社ドーン+京都大学)は**奈良市・尾張旭市・柏市の指令センターと連携**して導入。Coaido119とは正反対のアプローチで生き残っている。
  - 出典: [株式会社ドーン AED GO!](https://www.dawn-corp.co.jp/service/aed-go/) / [奈良市 AED GO!](https://www.city.nara.lg.jp/site/shobo-kyukyu/193059.html) / [inoti-aed.com](https://inoti-aed.com/guide/aed-go/)

### 3.5 e. 法的リスク
- ★日本には「**善きサマリア人の法 (good samaritan law) の明文化が弱い**」という構造課題があり、駆けつけた救助者の心理的ハードルが高い。
- [推測] 個人情報(医療資格情報)の本人確認運用、心停止情報という機微データの扱いは運営工数を圧迫した可能性大。
- [不確定] PMDA医療機器プログラム該当性は検索で言及記事を発見できず — おそらく該当しない判断で運営していた。

### 3.6 f. 技術・運営工数
- 24/365のサーバー稼働必須 (心停止対応で数秒の遅延が致命的)。
- 受信側ボランティアの**本人確認運用** (医療資格・CPR講習修了の検証) は**手作業前提** → スケール阻害。
- iOS のみ対応 → Android対応に追加投資できず受信側プールが頭打ち。
- ★**開発・運用コストは増え続ける一方、収益チャネルゼロ** → 営利会社で持続不可能。

## 6. 設計への教訓 (★ 最重要)

これから類似アプリを作る場合、Coaido119から学ぶべき**避けるべき判断**。

### 6.1 ★「119番並走モデル」を選ばない、指令センター連携を最初から組み込む
Coaido119の最大の構造的問題は**消防の指令センターと統合されていなかった**こと。「119番にダイヤルしつつアプリも押してね」は二段操作で実運用に乗らない。AED GO! の成功事例にあるように、**心停止疑いを指令センターが判定 → アプリへ自動配信** が正規ルート。これがない限り公的位置づけ(=自治体予算)を獲得できない。

### 6.2 ★営利会社単独で「無料の救命インフラ」を運営しない
Coaido株式会社は2025年3月解散。ボランティア依存・無料アプリ・24/365稼働必須・収益ゼロは**営利の体力では持続不可能**。最低でも以下のいずれか:
- (a) 一般社団・NPOで運営 + 自治体補助金/寄付/CSR予算
- (b) PulsePointモデルの財団化 + 各自治体年契約
- (c) 既存大企業 (SECOM/ALSOK等) のサービス内に組み込む

Coaidoは(a)に近い形でファストエイドへ移行したが、移行が遅すぎた可能性。

### 6.3 ★Two-sided市場の片側「受信者」入口を狭めない
Coaido119は「**iOS限定 + 18歳以上 + 医療資格/CPR講習修了**」と受信側のハードルが三重に高かった。受信側が薄い → SOSを打っても誰も来ない → 発見側がアンインストール、の負スパイラル。
- Android対応は最初からマスト。
- 18歳未満であっても (高校生でCPR講習修了者は多い) 登録可能にする (法的検討は必要だが)。
- 資格チェックは段階制 (ライト登録 → 検証アップグレード) で入口を広く。

### 6.4 ★実救命実績データを最初から計測・公表する設計に
Coaido119は実心停止現場での救命寄与の数字を公開できず、自治体・投資家への説得材料を欠いた。最初から **(a) SOS発火件数, (b) 通知到達率, (c) 承諾率, (d) 現着率, (e) ROSC寄与, (f) 神経学的予後** をログ収集し、myResponderの論文方式で定期公表する仕組みを設計に組み込むこと。

### 6.5 ★地理密度ファースト戦略 — 「全国対応」を急がない
Coaidoは豊島区実証直後に全国対応した結果、**どこに行っても登録者がスカスカ**になった。myResponderが小国シンガポールで成功した理由は密度。日本でやるなら**1区/1市単位で密度70%以上を達成してから次の自治体へ**。「全国対応です」は無意味で、**「この自治体内では半径500m以内に必ず登録者がいる」**を売る。

### 6.6 ★AEDマップは「自前構築」せず公的データに乗る
Coaidoは自前のAED情報(56,721件)を整備したが、AEDの移設・撤去・電池切れ追跡は不可能で常に古くなる。日本救急医療財団の[全国AEDマップ](https://www.jrc-cpr.org/aed-map/)など公的・既存DBに連動するか、AEDの所有者(自治体・施設)側に更新インセンティブを作るかしないと、UI上に表示されたAEDが現地で使えないリスクが残る。

### 6.7 ★「AEDエリアコール」のような自動架電機能は法務リスク・運営コスト過大
施設電話を勝手に鳴らす機能は、**全施設の事前同意が必要**で全国スケールに対して致命的にスケールしない (玄正氏自身が認めている)。設計するなら自動架電ではなく**プッシュ通知のみ**+施設管理者がアプリで応諾する設計。

### 6.8 ★最初からマネタイズチャネルを設計に内包する
「無料で広がってからビジネス化」は救命領域では成立しない (高齢化のニッチで広告モデルも厳しい)。当初から:
- B2G: 自治体年契約 (指令センター統合付き、PulsePointモデル)
- B2B: 大規模施設(タワマン/ショッピングモール/工場)の AED運用と組み合わせて法人契約
- B2B保険: 生命保険会社の付帯サービス (Coaidoが構想したが実装未到達)
のいずれかを最初の12-18ヶ月で立ち上げる前提で設計する。

### 6.9 ★ボランティア救助者の法的保護 (善きサマリア人の法) を運営側が補完する
日本では救助者免責が明文化されていない。アプリ運営側が:
- 利用規約で救助者の免責を明示
- 救助者向け保険を会社負担で付与
- 「来ました」ログだけ残し医療行為の良し悪しは問わない設計
を最初からやらないと、いざ呼ばれた時に駆けつける動機が崩れる。

### 6.10 ★「アプリ単独」ではなく「教育キット+アプリ」の二段構え
Coaidoの最終形態 = ファストエイドのCPRトレーニングボトルへの実質ピボットが示唆するのは、「**アプリだけでは救助者プールは増えない、現実の訓練機会と組み合わせて初めて機能する**」こと。設計初期から CPR 訓練イベント運営 / 訓練キット販売 / 講習動画 を同梱し、アプリは「訓練修了後の実戦投入装置」として位置づける。
